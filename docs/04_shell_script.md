## 1. テキストエディタ

シェルスクリプトを書くには、テキストファイルを作成・編集する**テキストエディタ**が必要です。

### 代表的なターミナル上のエディタ

UNIXのターミナルで使えるエディタとして、以下の3つが有名です。

| エディタ | 特徴 |
|---------|------|
| **Vim** | 高機能で広く普及。独特の操作体系（モード切替）を持つ |
| **Emacs** | 高機能で拡張性が高い。多くのキーボードショートカットを使う |
| **Nano** | シンプルで初心者にやさしい。画面下部に操作ガイドが表示される |

### Nano の基本操作

`nano` はターミナル上で動作する軽量なエディタです。操作方法が画面下部に表示されるため、初めてでも迷いにくいのが特徴です。

```bash
$ nano hello.sh     # hello.sh を作成・編集
```

画面下部に `^O Write Out`（保存）、`^X Exit`（終了）などのガイドが表示されます。`^` は Ctrl キーを意味します。

| 操作 | キー |
|------|------|
| 保存 | `Ctrl + O` → Enter |
| 終了 | `Ctrl + X` |
| カーソル移動 | 矢印キー |
| 行の切り取り | `Ctrl + K` |
| 貼り付け | `Ctrl + U` |

### 本講座での方針

本講座では、JupyterHub の**GUIエディタ** を使ってスクリプトを作成します。UNIXの扱いに不慣れだとCUIエディタでつまづくことが多いからです。時間があったら `nano` を試してみてください。

## 2. シェルスクリプトとは

これまで学んだコマンドは、ターミナルに1行ずつ入力して実行してきました。コマンドをテキストファイルにまとめたものが**シェルスクリプト**です。

| 利点 | 説明 |
|------|------|
| 再現性 | 同じスクリプトを実行すれば同じ結果が得られる |
| 記録 | 「何をどの順序で実行したか」がファイルとして残る |
| 自動化 | 複数のステップを一度に実行できる |
| 共有 | 解析手順を他の研究者と共有できる |

## 3. 最初のシェルスクリプト

### シバン（shebang）

シェルスクリプトの**1行目**には、このスクリプトをどのシェルで実行するかを指定する**シバン**（shebang）を書きます。

```bash
#!/bin/bash
```

`#!` に続けて、使用するシェルのパスを指定します。bash を使う場合は `#!/bin/bash` と書きます。

### スクリプトの作成と実行

JupyterHub でテキストエディタを開き、ファイル名を `hello.sh` に変更します。そして、以下の内容を入力して保存します。

```bash
#!/bin/bash
echo "Hello, Bioinformatics!"
echo "Today is $(date)"
```

> **`echo` とは**: 文字列を画面（標準出力）に表示するコマンドです。
>
> **`$(コマンド)` とは**: コマンドの実行結果をその場に展開する記法を**コマンド置換**と呼びます。`$(date)` なら `date` コマンドの出力（現在時刻）が埋め込まれます。

ターミナルでスクリプトを実行します。

```bash
# 方法1: bash コマンドで実行
$ bash hello.sh
Hello, Bioinformatics!
Today is 2026年 2月28日 土曜日 17時14分05秒 JST

# 方法2: 実行権限を付けて実行
$ chmod u+x hello.sh
$ ./hello.sh
```

> **`chmod u+x` とは**: ファイルに**実行権限**（eXecute permission）を付与するコマンドです。これにより `./ファイル名` で直接実行できるようになります。

## 4. 変数

### 変数の定義と使用

シェルスクリプトでは、**変数**を使って値に名前を付けることができます。変数を使うと、スクリプトの見通しが良くなり、設定の変更も簡単になります。

```bash
#!/bin/bash

GREETING="Hello"
NAME="Bioinformatics"

echo "${GREETING}, ${NAME}!"
```

> ⚠️ **注意**: `=` の前後に**スペースを入れてはいけません**。`NAME = "value"` はエラーになります。

### クォートの違い

ダブルクォートとシングルクォートでは、変数の展開に関して動作が異なります。

| クォート | 変数展開 | 使いどころ |
|---------|---------|-----------|
| `"..."` ダブルクォート | される | 変数の値を埋め込みたいとき（通常はこちら） |
| `'...'` シングルクォート | されない（すべてリテラル） | 文字列をそのまま渡したいとき |

```bash
NAME="World"
echo "Hello, ${NAME}!"   # → Hello, World!
echo 'Hello, ${NAME}!'   # → Hello, ${NAME}!
```

### 引数

スクリプト実行時に値を渡すことができます。これを**引数**（ひきすう）と呼びます。

```bash
$ bash myscript.sh input.fasta output.txt
```

スクリプト内では `$1`、`$2` … で受け取ります。

```bash
#!/bin/bash

INPUT=$1
OUTPUT=$2

echo "Input:  ${INPUT}"
echo "Output: ${OUTPUT}"
```

| 特殊変数 | 意味 |
|---------|------|
| `$1`, `$2`, … | 1番目、2番目の引数 |
| `$0` | スクリプト自身のファイル名 |
| `$#` | 引数の個数 |

## 5. コメント

`#` から行末まではコメントとして扱われ、実行されません（ただし1行目のシバンは例外です）。

```bash
#!/bin/bash
# Usage: bash run_analysis.sh input.fasta output.txt

INPUT=$1
OUTPUT=$2
```

### コメントを書く言語について

本講義の資料では、日本人学生にわかりやすく伝えるため基本的に日本語でコメントを書いています。しかし、**実際の研究で用いるスクリプトでは英語でコメントを書くのが標準的**です。学術誌へ論文を投稿する際、解析に使用したスクリプトはGitHubなどで公開する必要があります。普段から英語で書く習慣をつけましょう。

## 6. ループ：for文

### 基本構文

`for` 文は、リストの各要素に対して同じ処理を繰り返します。

```bash
for 変数名 in リスト; do
    処理
done
```

### 例1：リストのループ

```bash
#!/bin/bash

for SAMPLE in human mouse zebrafish; do
    echo "Processing ${SAMPLE}..."
done
```

```
Processing human...
Processing mouse...
Processing zebrafish...
```

### 例2：連番のループ

`$(seq 開始 終了)` で連番のリストを生成できます。`$(seq 1 5)` は `1 2 3 4 5` に展開されます。

```bash
#!/bin/bash

for i in $(seq 1 5); do
    echo "This is sample ${i}" > sample_${i}.txt
done
```

これを実行すると `sample_1.txt`〜`sample_5.txt` が作成されます。

## 7. 条件分岐：if文

### 基本構文

`if` 文は、条件が成り立つかどうかで処理を分岐します。

```bash
if [ 条件 ]; then
    条件が真のときの処理
else
    条件が偽のときの処理
fi
```

> ⚠️ **注意**: `[` と条件の間、条件と `]` の間には**スペースが必要**です。`[条件]` はエラーになります。

`elif`（else if）を使うと、条件を追加できます。

```bash
if [ 条件1 ]; then
    条件1が真のときの処理
elif [ 条件2 ]; then
    条件2が真のときの処理
else
    どちらも偽のときの処理
fi
```

### よく使う条件式

#### ファイルに関する条件

| 条件 | 意味 |
|------|------|
| `-f ファイル` | 通常ファイルが存在する |
| `-d ディレクトリ` | ディレクトリが存在する |

#### 文字列に関する条件

| 条件 | 意味 |
|------|------|
| `"$A" = "$B"` | 文字列が等しい |
| `"$A" != "$B"` | 文字列が等しくない |

#### 数値に関する条件

| 条件 | 意味 |
|------|------|
| `$A -eq $B` | 等しい（equal） |
| `$A -ne $B` | 等しくない（not equal） |
| `$A -gt $B` | より大きい（greater than） |
| `$A -lt $B` | より小さい（less than） |

### 例1：ファイルの存在チェック

```bash
#!/bin/bash

INPUT="proteins.fasta"

if [ -f ${INPUT} ]; then
    echo "${INPUT} found."
else
    echo "Error: ${INPUT} not found!"
fi
```

### 例2：for と if の組み合わせ

```bash
#!/bin/bash

for SAMPLE in human mouse zebrafish; do
    if [ "${SAMPLE}" = "human" ]; then
        echo "${SAMPLE}: target species"
    else
        echo "${SAMPLE}: outgroup"
    fi
done
```

## 8. 実践：BLAST解析スクリプトを例に

オプションが多いコマンドは1行にまとめると読みにくくなります。行末に `\` を置くと「次の行に続く」という意味になり、コマンドを複数行に分けて書けます。

前のセクション（[BLAST初級](./03_blast.md)）で扱ったBLASTコマンドも、`\` を使うとすっきりまとめられます。

```bash
# 1行で書いた場合
blastp -query query.fasta -db db -out query_vs_db.fmt6.txt -outfmt 6

# \ で分割した場合（同じコマンド）
blastp \
  -query query.fasta \
  -db db \
  -out query_vs_db.fmt6.txt \
  -outfmt 6
```

では、シェルスクリプトに書き直してみましょう。

```bash
#!/bin/bash

#=== CONFIGS ===
QUERY="query.fasta"
DB_NAME="db"
FMT=6
OUTPUT="query_vs_db.fmt${FMT}.txt"
#===

blastp \
  -query ${QUERY} \
  -db ${DB_NAME} \
  -out ${OUTPUT} \
  -outfmt ${FMT}
```

このスクリプトを別のクエリやデータベースに転用したいときは、冒頭の変数（`QUERY`、`DB_NAME` など）を書き換えるだけで済みます。`blastp` のコマンド本体には手を加える必要はありません。
