## 1. ターミナルとシェル

### ターミナルとは

**ターミナル**（端末）は、コンピュータに文字を入力してコマンドを実行するためのインターフェースです。GUIアプリのようにマウスでクリックするのではなく、**キーボードでコマンド（命令）を打ち込む**ことでコンピュータを操作します。

### シェルとは

**シェル**は、ユーザーが入力したコマンドをOSに橋渡しするプログラムです。様々な種類がありますが、本講座では **bash**（Bourne Again SHell）を使います。

JupyterHub では、上部メニューから **File → New → Terminal** を選択するとターミナルが起動します。

```bash
$ echo "Hello, Bioinformatics!"
Hello, Bioinformatics!
```

> **`$` マークについて**: ターミナルに表示される `$`（プロンプト）はコマンドの入力待ち状態を示します。`$` 自体は入力しません。

## 2. ディレクトリ構造とパス

### ディレクトリとは

コンピュータのディスク上の記憶領域は、**ディレクトリ**（フォルダ）と呼ばれる「部屋」に分けて管理されています。ディレクトリの中にはファイルや別のディレクトリを入れることができ、全体として**階層構造（ツリー構造）**を形成しています。

この階層の頂点に位置するディレクトリを**ルートディレクトリ**（`/`）といいます。そこから枝分かれするように、さまざまなディレクトリ・ファイルが配置されています。

| 記号 | 名称 | 内容 |
|------|------|------|
| `/`  | ルートディレクトリ | 階層の最上位にあるディレクトリ |
| `~`  | ホームディレクトリ | ログイン時に最初に位置するディレクトリ（`/home/ユーザ名`） |
| `.`  | カレントディレクトリ（ワーキングディレクトリ） | 現在作業中のディレクトリ |
| `..` | 親ディレクトリ | カレントディレクトリの1つ上のディレクトリ |

### ディレクトリ構造の例

以下は `/home/shunta/` 以下のディレクトリ・ファイルの例です。

```
/home/shunta/
├── memo.txt
└── project01/
    ├── analysis/
    ├── data/
    │   ├── features.gff3
    │   └── sequences.fasta
    └── script/
        └── run_tool.sh
```

### パスとは

ファイルやディレクトリの**場所を示す文字列**のことを**パス**（path）といいます。住所のようなもので、どの階層にあるかを `/`（スラッシュ）でつなぎながら表現します。

例えば、上の図の `run_tool.sh` の場所は次のように表せます：

```
/home/shunta/project01/script/run_tool.sh
```

これは「ルートディレクトリ → `home` → `shunta` → `project01` → `script` → `run_tool.sh`」という経路を表しています。

### 絶対パスと相対パス

パスの書き方には2種類あります。

**絶対パス**: ルートディレクトリ `/` を起点とした経路
```
/home/shunta/project01/data/sequences.fasta
```

**相対パス**: カレントディレクトリを起点とした経路
```
project01/data/sequences.fasta    ← カレントディレクトリが /home/shunta/ のとき
../data/sequences.fasta           ← カレントディレクトリが /home/shunta/project01/script/ のとき
```

> **どちらを使う？**: 絶対パスはどこにいても通じる確実な書き方です。相対パスは短く書けて便利ですが、カレントディレクトリが変わると書き方を変更する必要があります。

ホームディレクトリを起点としてパスを指定することも可能です。
```
~/memo.txt
~/project01/data/sequences.fasta
```

## 3. 基本コマンド

### コマンドの入力方法

UNIXのコマンドは、基本的に次の形式で入力します。

```
コマンド名 [オプション] [引数]
```

- **オプション**: コマンドの動作を変えるための指定。`-` に続けて1文字で書くことが多い
- **引数**: コマンドの処理対象（ファイル名やディレクトリ名など）

オプションは複数指定でき、`-l -a` のようにスペースで区切るか、`-la` のようにまとめて書くこともできます。以下に例を示しますが、`ls` コマンドそのものについては後述します。

```bash
$ ls              # 引数なし・オプションなし
$ ls data         # 引数あり（data ディレクトリの中身を表示）
$ ls -l           # オプションあり（詳細表示）
$ ls -a           # オプションあり（隠しファイルも表示）
$ ls -la          # 複数オプションをまとめて指定（-l と -a の組み合わせ）
```

### カレントディレクトリの表示：`pwd`

**pwd** (Print Working Directory) は、カレントディレクトリを表示します。

```bash
$ pwd
/home/shunta
```

### ディレクトリ内容の表示：`ls`

**ls** (LiSt) は、ディレクトリの中身を一覧表示します。

```bash
$ ls
memo.txt  project01

$ ls project01       # 引数にディレクトリを指定すると、その中身を表示
analysis  data  script

$ ls -l              # 詳細情報を表示（権限・サイズ・更新日時など）
$ ls -lh             # ファイルサイズを読みやすく表示（K, M, G など）
$ ls -a              # 隠しファイル（. で始まるファイル）も表示
```

### カレントディレクトリの移動：`cd`

**cd** (Change Directory) で指定ディレクトリに移動します。

```bash
$ cd project01       # project01 ディレクトリへ移動
$ cd ..              # 親ディレクトリへ移動
$ cd ~               # ホームディレクトリへ移動
$ cd                 # 同上（引数なしでもホームに戻る）
```

### ディレクトリの作成：`mkdir`

**mkdir** (MaKe DIRectory) でディレクトリを作ります。

```bash
$ mkdir project01
$ mkdir analysis data script           # 複数のディレクトリをまとめて作成
$ mkdir -p project02/data/raw          # 中間ディレクトリも含めて一括作成
```

### ファイルのコピー：`cp`

**cp** (CoPy) でファイルをコピーします。

```bash
$ cp sequences.fasta sequences_backup.fasta    # ファイルをコピー
$ cp -r data/ data_backup/                     # ディレクトリごとコピー（-r = recursive）
```

> ⚠️ **注意**: コピー先に同名ファイルがあると上書きされます。

### ファイルの移動・名前変更：`mv`

**mv** (MoVe) でファイルを移動したり名前を変えたりします。

```bash
$ mv result.txt analysis/result.txt    # analysis/ ディレクトリへ移動
$ mv old_name.txt new_name.txt         # ファイル名を変更
```

### ファイルの削除：`rm`

**rm** (ReMove) でファイルを削除します。

```bash
$ rm temp.txt                 # ファイルを削除
$ rm -r old_results/          # ディレクトリごと削除
```

> ⚠️ **注意**: `rm` で削除したファイルは**ゴミ箱には入らず完全に消えます**。復元はできないので慎重に。

## 4. ワイルドカード

ワイルドカードを使うと、**複数のファイルをまとめて指定**できます。

| 記号 | 意味 | 例 |
|------|------|-----|
| `*` | 任意の文字列（0文字以上） | `*.fasta` → 全FASTAファイル |
| `?` | 任意の1文字 | `seq?.fasta` → seq1.fasta, seq2.fasta など |
| `[abc]` | 括弧内のいずれか1文字 | `seq[12].fasta` → seq1.fasta, seq2.fasta |

```bash
# data/ ディレクトリ内の全FASTAファイルを一覧表示
$ ls data/*.fasta
```

## 5. ターミナルの便利な操作

実際にコマンドを入力するとき、毎回すべてを手打ちする必要はありません。以下の操作を積極的に活用しましょう。

### Tab補完

コマンドやファイル名の途中で **Tabキー** を押すと自動補完されます。

```bash
$ cd unix_pra[Tab]
$ cd unix_practice/     ← 自動補完！
```

候補が複数ある場合は **Tab を2回** 押すと候補一覧が表示されます。

> **Tips**: スペルミスによるエラーを防ぐためにも、Tab補完を積極的に使いましょう。

### カーソル移動と編集

以下のショートカットを使うと効率よく編集できます。

| キー操作 | 動作 |
|---------|------|
| `←` / `→` | カーソルを1文字移動 |
| `Ctrl+A` | 行の先頭へ移動 |
| `Ctrl+E` | 行の末尾へ移動 |
| `Ctrl+U` | カーソルより前を全削除 |
| `Ctrl+K` | カーソルより後を全削除 |
| `Ctrl+Y` | 直前に削除したテキストを貼り付け |
| `Ctrl+L` | 画面をクリア |

### コマンド履歴

`↑` / `↓`（または `Ctrl+P` / `Ctrl+N`）を押すと、以前に入力したコマンドを呼び出せます。

> **Tips**: 長いコマンドを再利用するときは `↑` で呼び出し、一部だけ修正すると効率的です。

### コマンドの強制終了

**`Ctrl+C`** を押すと、実行中のコマンドを強制終了できます。重い処理を誤って実行してしまったときや、プログラムが応答しなくなったときに使います。

## 6. ファイルの中身を見る

### ファイル全体を表示：`cat`

**cat** (conCATenate) でファイルの中身を表示します。短いファイルの確認に向いています。

```bash
$ cat sequences.fasta
>NP_001119606.1 membrane alanyl aminopeptidase N precursor [Acyrthosiphon pisum]
MFKLVAVLSAALLCALVRGQEDADDKKMDVFRLPENTSPESYDLWFAPNMNDWTFEGCAKILVNINTPDT...
```

### 先頭・末尾だけ見る：`head` / `tail`

大きなファイルをざっと確認したいときに使います。

```bash
$ head sequences.fasta          # 先頭10行を表示（デフォルト）
$ head -n 5 sequences.fasta     # 先頭5行を表示
$ tail sequences.fasta          # 末尾10行を表示（デフォルト）
$ tail -n 20 sequences.fasta    # 末尾20行を表示
```

### スクロールして見る：`less`

**less** は大きなファイルをページ送りしながら閲覧するコマンドです。TSVファイルなど横に長いファイルを見るときは `-S` オプションで行の折り返しを無効にすると見やすくなります。

```bash
$ less sequences.fasta
$ less -S features.gff3    # 行を折り返さずに表示（横スクロールで確認できる）
```

| キー | 動作 |
|------|------|
| `f` / `Space` | 次のページへ |
| `b` | 前のページへ |
| `↑` / `↓` | 1行ずつスクロール |
| `←` / `→` | 左右にスクロール（`-S` 使用時） |
| `g` | ファイルの先頭へ移動 |
| `G` | ファイルの末尾へ移動 |
| `/文字列` | 文字列検索 |
| `n` | 次の検索結果へ |
| `q` | 終了 |

## 7. 圧縮・展開：gzip と tar

公開されているゲノムデータやシーケンスデータは、多くの場合圧縮された形式で配布されます。

### gzip による圧縮・展開

```bash
$ gzip sequences.fasta          # 圧縮 → sequences.fasta.gz が作成され、元ファイルは消える
$ gunzip sequences.fasta.gz     # 展開 → sequences.fasta が復元される
$ zless sequences.fasta.gz      # 展開せずに中身を確認（less と同じ操作）
```

> **Tips**: `zless` は `.gz` ファイルを展開せずに閲覧できるため、大きなファイルを確認するときに便利です。

### tar によるアーカイブ

複数のファイルをひとまとめにして圧縮します。

```bash
# 圧縮（data/ ディレクトリをまとめて archive.tar.gz に）
$ tar -czf archive.tar.gz data/

# 複数のファイルをまとめて圧縮（ディレクトリではなくファイルを列挙）
$ tar -czf archive.tar.gz file1.fasta file2.fasta file3.gff

# 展開
$ tar -xzf archive.tar.gz

# 中身の確認（展開せずに）
$ tar -tzf archive.tar.gz
```

| オプション | 意味 |
|-----------|------|
| `-c` | アーカイブを作成（create） |
| `-x` | アーカイブを展開（extract） |
| `-z` | gzip 圧縮を使用 |
| `-f` | ファイル名を指定 |
| `-v` | 詳細を表示（verbose） |

## 8. データ取得：curl と wget

バイオインフォマティクスでは、データベースからコマンドラインで直接データをダウンロードする場面が多くあります。

### curl

**curl** は URL を指定してデータを取得するコマンドです。オプションなしで実行すると、取得した内容が標準出力（画面）に表示されます。ファイルとして保存したい場合はオプションを指定します。

```bash
# オプションなし → 内容が画面に表示される
$ curl https://example.com/data/sequences.fasta

# -o（小文字）: 指定したファイル名で保存
$ curl -o my_sequences.fasta https://example.com/data/sequences.fasta

# -O（大文字）: URL末尾のファイル名のまま保存（この例では sequences.fasta として保存）
$ curl -O https://example.com/data/sequences.fasta
```

### wget

**wget** も URL からファイルをダウンロードするコマンドです。`curl` と異なり、オプションなしでも自動的にファイルとして保存されます。

```bash
# オプションなし → URL末尾のファイル名で保存（この例では sequences.fasta として保存）
$ wget https://example.com/data/sequences.fasta

# -O（大文字）: 指定したファイル名で保存
$ wget -O my_sequences.fasta https://example.com/data/sequences.fasta
```

## 9. コマンドの調査：man と which

### コマンドのマニュアルを表示：`man`

**man** (MANual) は、コマンドの公式マニュアルを表示します。コマンドの使い方を知るのに便利です。

```bash
$ man ls       # ls コマンドのマニュアルを表示
```

マニュアルの操作は `less` と同じです（スペースで次ページ、`q` で終了）。

> **Tips**: Web検索やAIに聞く前にまず `man` で一次情報を確認する習慣をつけると力がつきます。

### コマンドのパスを表示：`which`

**which** は、指定したコマンドがどこにインストールされているかを表示します。

```bash
$ which less
/usr/bin/less
```
